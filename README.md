## 1. 平台概述

### 1.1 什么是 Study Clash？

**Study Clash** 是一个**游戏化学习对战平台**，核心理念是"让学习变得更有趣，让竞争激发潜能"。平台将学习与竞技结合，通过多种游戏模式激励学生主动学习，同时提供 AI 驱动的智能分析和个人化学习推荐。

### 1.2 核心功能

| 功能模块             | 说明                                                   |
| -------------------- | ------------------------------------------------------ |
| 🎮 **多人实时对战**   | 单人挑战、双人对战、四人挑战三种模式                   |
| 📚 **智能错题管理**   | 基于遗忘曲线的间隔重复复习系统                         |
| 🤖 **AI 学习助手**    | 归因分析、内容生成、学习策略、智能问答等 12 项 AI 功能 |
| 🏆 **游戏化段位系统** | 8 大段位、24 级晋升体系，排行榜竞争                    |
| 👨‍🏫 **多角色权限**    | 管理员、教师、学生三种角色，细粒度权限控制             |
| 📊 **数据分析**       | 学习统计、知识点掌握图谱、班级学情分析                 |

### 1.3 技术架构

| 层级          | 技术栈                                    |
| ------------- | ----------------------------------------- |
| 后端框架      | Flask 3.0 + Python 3.11+                  |
| 数据库        | PostgreSQL 15+                            |
| 缓存/消息队列 | Redis 7+                                  |
| 实时通信      | Socket.IO (WebSocket)                     |
| 服务器        | Gunicorn + Eventlet (32 Workers)          |
| ORM           | SQLAlchemy 2.0 + Alembic                  |
| 前端          | Bootstrap 5 + Jinja2 + ECharts + Chart.js |
| 部署          | Docker + Docker Compose                   |

### 1.4 系统要求

| 项目     | 最低配置                          | 推荐配置           |
| -------- | --------------------------------- | ------------------ |
| CPU      | 2 核                              | 4 核+              |

## 3. 系统安装与部署

### 3.1 本地开发环境（两种方案）

#### 方案 1：使用 Docker（推荐）

```bash
# 1. 启动本地 PostgreSQL 容器
docker run -d \
  --name studyclash-db-dev \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  postgres:15-alpine

# 2. 等待数据库就绪（约 10 秒）
docker logs -f studyclash-db-dev
# 看到 "database system is ready to accept connections" 即可

# 3. 进入项目目录
cd "c:\Users\Joner\Desktop\Study Clash\Study Clash"

# 4. 激活虚拟环境（Windows）
.venv\Scripts\Activate.ps1

# 5. 初始化数据库
flask db upgrade

# 6. 启动服务
python run.py
```

#### 方案 2：安装本地 PostgreSQL

```bash
# 1. 安装 PostgreSQL 15+
# Windows: https://www.postgresql.org/download/windows/

# 2. 创建数据库
psql -U postgres -c "CREATE DATABASE studyclash_dev;"

# 3. 进入项目目录并启动
cd "c:\Users\Joner\Desktop\Study Clash\Study Clash"
.venv\Scripts\Activate.ps1
flask db upgrade
python run.py
```

### 3.2 生产环境部署（Docker Compose）

#### 步骤 1：安装 Docker

```bash
curl -fsSL https://get.docker.com | sudo bash
sudo usermod -aG docker $USER
newgrp docker
docker --version
docker compose version
```

#### 步骤 2：获取项目

```bash
git clone https://github.com/your-username/study-clash.git
cd study-clash
```

#### 步骤 3：配置环境变量

```bash
chmod +x scripts/setup-env.sh
./scripts/setup-env.sh
```

脚本自动完成：

- ✅ 生成随机 SECRET_KEY
- ✅ 生成随机数据库密码
- ✅ 生成 LLM_ENCRYPTION_KEY
- ✅ 设置 FLASK_ENV=production

#### 步骤 4：启动服务

```bash
docker compose up -d
docker compose logs -f app
# 等待看到: Listening at: http://0.0.0.0:5002
```

#### 步骤 5：初始化数据库

```bash
docker exec studyclash-app flask db upgrade
docker exec studyclash-app flask init-db
```

#### 步骤 6：配置 HTTPS（推荐）

```bash
# 安装 Caddy
sudo apt install caddy

# 配置 Caddy（自动获取 SSL 证书）
echo "your-domain.com { reverse_proxy localhost:80 }" | sudo tee /etc/caddy/Caddyfile
sudo systemctl enable caddy && sudo systemctl start caddy
```

### 3.3 升级更新

```bash
# 1. 拉取最新代码
git pull

# 2. 检查并修复配置
./scripts/setup-env.sh

# 3. 重建并重启
docker compose up -d --build

# 4. 执行数据库迁移
docker exec studyclash-app flask db upgrade
```

### 3.4 常用命令

```bash
# 服务管理
docker compose up -d          # 启动
docker compose down           # 停止（保留数据）
docker compose restart        # 重启
docker compose ps             # 查看状态

# 日志查看
docker compose logs -f app    # 实时日志
docker compose logs --tail=100 app  # 最近 100 行

# 数据库操作
docker exec -it studyclash-db psql -U postgres -d studyclash
docker exec studyclash-app flask db current
docker exec studyclash-app flask db upgrade

# 重置管理员密码
docker compose exec -it app flask reset-admin-password
```

## 4. 账号注册与登录

### 4.1 注册新账号

1. 点击首页右上角 **"注册"** 按钮
2. 填写信息：

**必填字段**：

| 字段     | 说明                                       |
| -------- | ------------------------------------------ |
| 用户名   | 唯一，用于登录（实时检查是否被占用）       |
| 邮箱     | 唯一，用于登录和通知（实时检查是否已注册） |
| 密码     | 需符合当前密码策略                         |
| 确认密码 | 与密码一致                                 |

**选填字段**：

| 字段 | 说明                           |
| ---- | ------------------------------ |
| 昵称 | 显示名称，不填则默认使用用户名 |
| 年级 | 从字典选项选择                 |
| 学校 | 从字典选项选择                 |

3. 点击 **"注册"** 完成
4. 系统自动登录并跳转首页

> 💡 **注册限流**: 同一 IP 每 60 秒最多 3 次注册请求

### 4.2 登录

1. 点击 **"登录"** 按钮
2. 输入用户名（或邮箱）和密码
3. 可勾选 **"记住我"** 延长会话
4. 点击登录

**登录结果提示**：

| 情况             | 系统提示                      |
| ---------------- | ----------------------------- |
| 登录成功         | "欢迎回来，[昵称]！"          |
| 用户名或密码错误 | "用户名或密码错误"            |
| 账户被锁定       | "账户已锁定，请 X 分钟后重试" |
| 账户被禁用       | "账户已被禁用"                |
| 需修改默认密码   | 强制跳转密码修改页            |

> ⚠️ **账户锁定规则**: 连续 5 次登录失败后，账户锁定 30 分钟（管理员豁免）  
> ⚠️ **登录限流**: 每 60 秒最多 5 次登录尝试

### 4.3 个人资料管理

**可修改的信息**：

| 字段   | 说明                     |
| ------ | ------------------------ |
| 昵称   | 修改显示名称             |
| 年级   | 修改年级                 |
| 学校   | 修改学校                 |
| 新密码 | 修改密码（留空则不修改） |

**修改密码流程**：

1. 输入原密码
2. 输入新密码和确认新密码
3. 新密码需符合当前密码策略
4. 点击保存

| 情况         | 效果                                       |
| ------------ | ------------------------------------------ |

## 5. 游戏模式详解

### 5.1 三种游戏模式

| 模式           | 玩家数 | 说明                         |
| -------------- | ------ | ---------------------------- |
| 🎯 **单人挑战** | 1 人   | 独自答题，3 次错误机会       |
| ⚔️ **双人对战** | 2 人   | 实时 PK，比拼速度和正确率    |
| 🏆 **四人挑战** | 4 人   | 4 人同台竞技，按排名分配积分 |

### 5.2 单人挑战模式

**规则**：

- ❌ 默认 **3 次错误机会**，用完游戏结束
- ⏱️ 每题限时（默认 60 秒），超时视为错误
- 🏅 每答对一题获得积分（默认 1 分/题）

**操作步骤**：

1. 选择学科和章节（章节可选）
2. 系统随机抽取题目（默认 10 题）
3. 答题流程：
   - 显示题目，开始倒计时
   - 选择/输入答案 → 提交
   - 即时判断对错，显示正确答案和解析
   - 正确 → 得分 +1，继续下一题
   - 错误 → 错误次数 +1（未用完机会则继续，用完则结束）
   - 超时 → 视为错误
4. 结果页：显示总分、正确数、错误数、用时、获得积分
5. **再来一局**：使用相同参数直接开始新局

### 5.3 双人对战模式

**创建房间**：

1. 进入 **双人对战** 页面
2. 选择学科/章节
3. 点击 **"创建房间"**
4. 系统生成 **6 位房间码**（大写字母 + 数字）
5. 将房间码分享给对手
6. 等待对手加入

**加入房间**：

- 方式一：输入房间码加入
- 方式二：从房间列表选择

**准备与开始**：

1. 双方进入房间后，各自点击 **"准备"** 按钮
2. 两位玩家都准备后，游戏自动开始

**对战规则**：

1. 两人同时答题，题目相同
2. 实时看到对方答题进度（WebSocket）
3. 每题限时，先答对且正确的一方在该题得分
4. 答完所有题目后比较总分
5. **胜利方**获得 `battle_win` 积分（默认 10 分）

**聊天功能**：
在等待房间和游戏中，可以发送聊天消息与对手交流。

**再来一局**：

1. 游戏结果页面，一方点击 **"再来一局"** 发起邀约
2. 对手收到邀约通知，选择同意/拒绝
3. 邀约 **15 秒超时**：若 15 秒内未全部同意，邀约自动取消
4. 全部同意后，使用相同参数创建新房间并开始

**换个对手**：
点击 **"换个对手"**，保持相同参数创建新等待房间，等待新对手加入。

### 5.4 四人挑战模式

**积分分配**：

| 排名    | 积分项        | 默认积分 | 说明   |
| ------- | ------------- | -------- | ------ |
| 第 1 名 | `four_first`  | 30       | 🥇 冠军 |
| 第 2 名 | `four_second` | 20       | 🥈 亚军 |
| 第 3 名 | `four_third`  | 10       | 🥉 季军 |
| 第 4 名 | `four_fourth` | 5        | 第四名 |

**操作流程**：

1. 创建房间或加入房间（需等 4 人齐）
2. 4 人全部点击 **"准备"** 后游戏开始
3. 答题结束后按得分排名（正确数降序 → 用时升序）
4. 按排名分配积分
5. 再来一局需 4 人全部同意

### 5.5 断点续答与掉线重连

**断点续答**：

- 重新进入游戏页面时，系统自动检测进行中的游戏
- 从上次答题位置继续，已答题目不会重复出现

**掉线重连**：

- 系统通过 `/game/api/active-game` 检测进行中的游戏
- 通过 `/game/api/answered-questions/<room_code>` 获取已答题目信息
- 答题进度存储在 `GamePlayer.current_question_index` 和 `UserAnswer` 记录中

> 💡 掉线后尽快重连，游戏仍在进行中！

### 5.6 游戏历史

在 **游戏 → 历史记录** 页面可查看所有游戏记录：

| 信息          | 说明               |
| ------------- | ------------------ |
| 游戏类型      | 单人/双人/四人     |
| 学科          | 游戏学科           |
| 得分          | 游戏得分           |
| 正确数/错误数 | 答题正确和错误数量 |
| 用时          | 总答题用时         |
| 排名          | 在房间中的排名     |
| 获得积分      | 本次游戏获得的积分 |
| 时间          | 游戏时间           |

- 按时间倒序排列
- 每页 10 条，支持分页

## 6. 题目类型与答题指南

### 6.1 四种题型

| 题型       | 答题方式           | 正确答案格式           | 示例            |
| ---------- | ------------------ | ---------------------- | --------------- |
| **单选题** | 从 A-F 选择一个    | 单个字母               | `A`             |
| **多选题** | 从 A-F 选择多个    | 逗号分隔，完全匹配     | `A,B,C`         |
| **判断题** | 选择"正确"或"错误" | `T`/`F` 或 `对`/`错`   | `T`             |
| **填空题** | 输入框输入         | 支持 `\|` 分隔多个答案 | `北京\|Beijing` |

### 6.2 四个难度等级

| 难度值 | 难度名称 |
| ------ | -------- |
| 1      | 简单     |
| 2      | 中等     |
| 3      | 困难     |
| 4      | 极难     |

### 6.3 答题注意事项

**单选题**：

- 从 A-F 中选择**一个**选项
- 部分题目配有题干图片或选项图片
- 仔细查看图片内容后再作答

**多选题**：

- 必须**选中所有正确选项**才算对（完全匹配）
- 多选、少选、错选均判为错误
- 最少选 2 个选项

**判断题**：

- 选择 **"正确"** 或 **"错误"**

**填空题**：

- 答案忽略首尾空格（"  北京  " 和 "北京" 效果相同）
- 一个填空题可能有**多个可接受的正确答案**
- 多个正确答案用 `|` 分隔，但对用户不可见

### 6.4 题目字段说明

| 字段     | 说明                         |
| -------- | ---------------------------- |
| 题干内容 | 题目描述                     |
| 题干图片 | 可选，配合题干展示           |
| 选项 A-F | 文字选项（单选/多选/判断题） |
| 选项图片 | 可选，每个选项可配图         |
| 答案解析 | 答题后显示详细解析           |
| 题目分值 | 默认 10 分                   |
| 答题时限 | 默认 60 秒                   |

## 7. 积分与段位系统

| 获取方式     | 积分项           | 默认积分 | 说明           |

| 四人冠军     | `four_first`     | 30       | 第 1 名        |

| 每日登录     | `daily_login`    | 5        | 每天首次登录   |

> 💡 实际积分数值可被管理员调整

### 7.2 24 级段位体系

8 大段位 × 3 小段位 = 24 级：

| 感知等级 | 大段位 | 英文名   | 小段位       | 图标 |
| -------- | ------ | -------- | ------------ | ---- |
| 1-3      | 青铜   | Bronze   | III → II → I | 🔔    |
| 4-6      | 白银   | Silver   | III → II → I | 🔮    |
| 7-9      | 黄金   | Gold     | III → II → I | 🥇    |
| 10-12    | 铂金   | Platinum | III → II → I | 💎    |
| 13-15    | 钻石   | Diamond  | III → II → I | 💠    |
| 16-18    | 大师   | Master   | III → II → I | 🌟    |
| 19-21    | 王者   | King     | III → II → I | 👑    |
| 22-24    | 传说   | Legend   | III → II → I | ✨    |

**晋升规则**：

- 积分达到新段位门槛时**自动晋升**

## 8. 错题本与复习系统

### 8.1 错题自动收集

答错题目时，系统自动将错题加入错题本：

- 同一题不重复添加，只增加错误计数
- 记录信息：错误答案、错误次数、出错原因、游戏类型、时间
- 保留最近 **10 次** 错误记录历史

**错题原因标注**：

| 原因代码        | 中文说明     |
| --------------- | ------------ |
| `careless`      | 粗心大意     |
| `knowledge`     | 知识点不熟悉 |
| `understanding` | 审题错误     |
| `calculation`   | 计算错误     |
| `memory`        | 记忆模糊     |
| `other`         | 其他原因     |

**错题标记**：

- ⭐ **标记重要**：重要错题在复习时优先展示
- 📝 **添加笔记**：个人对错题的理解和反思

### 8.2 间隔重复复习（Spaced Repetition）

基于**遗忘曲线**智能安排复习时间。

**复习间隔递增表**：

| 连续正确次数 | 下次复习间隔 | 说明                            |
| ------------ | ------------ | ------------------------------- |
| 0            | 1 天         | 刚答错，短期复习                |
| 1            | 2 天         | 答对一次，间隔延长              |
| 2            | 4 天         | 连续两次正确                    |
| 3            | 7 天         | 连续三次正确 → **标记为已掌握** |
| 4            | 15 天        | 掌握后进一步延长                |
| 5            | 30 天        | 深度巩固                        |

**高频错题加速**：

- 错误次数 ≥ 5 的题目，首次复习间隔缩短为 **12 小时**

**掌握判定**：

- 连续正确 **3 次**标记为 **已掌握**
- 已掌握题目不再出现在复习列表中
- 复习中再次答错，连续正确计数重置为 0

**复习排序优先级**：

1. ⭐ 重要题目优先
2. 🔢 错误次数多优先
3. ⏰ 到期时间早优先

**每日复习上限**：默认每天最多 **50 题**

**复习积分**：每答对一题获得 `review_correct` 积分（默认 1 分）

### 8.3 收藏集

1. **创建收藏集**：填写名称、描述，设置是否公开
2. **添加错题**：将错题加入收藏集（不重复添加）
3. **添加笔记**：在收藏集内为错题添加个人笔记
4. **移除错题**：从收藏集中移除不需要的错题

### 8.4 闯关进度

- **每个章节 = 一个关卡**
- 通关条件：完成该章节所有题目且达到一定正确率
- 记录通关状态和通关时间
- 每个用户每个章节独立记录进度

### 8.5 学习打卡

- 每日进行错题复习即自动打卡
- 记录每日复习次数
- 可查看打卡日历和连续打卡天数

### 8.6 薄弱知识点分析

在错题本中查看：

- 按章节统计错误数量
- 显示未掌握题目数量
- 按错误数量降序排列（默认前 10 个）

### 8.7 相似题目推荐

查看某错题时，系统推荐相似题目：

- 推荐条件：**同学科 + 同章节 + 同题型 + 难度相近**
- 已掌握的相似题排在后面
- 默认推荐 **5 道** 相似题

# 创建备份脚本
cat > ~/backup_studyclash.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=~/studyclash_backup
mkdir -p $BACKUP_DIR

# 备份数据库
docker exec studyclash-db pg_dump -U postgres studyclash | gzip > $BACKUP_DIR/db_$(date +%Y%m%d_%H%M%S).sql.gz

# 备份上传文件
docker cp studyclash-app:/app/uploads $BACKUP_DIR/uploads_$(date +%Y%m%d) 2>/dev/null || true

# 删除 7 天前的备份
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete
find $BACKUP_DIR -type d -name "uploads_*" -mtime +7 -exec rm -rf {} + 2>/dev/null || true

echo "备份完成: $(date)"
EOF

chmod +x ~/backup_studyclash.sh

(crontab -l 2>/dev/null; echo "0 2 * * * ~/backup_studyclash.sh") | crontab -

## 20. 安全管理

### 20.1 登录锁定机制

| 配置项 Key           | 默认值 | 范围 | 说明             |
| -------------------- | ------ | ---- | ---------------- |
| `max_login_attempts` | 5      | 3-10 | 最大连续失败次数 |
| `lockout_duration`   | 30     | 5-60 | 锁定时长（分钟） |

**工作流程**：

1. 用户登录失败 → `LoginAttempt.fail_count` 加 1
2. 当 `fail_count >= max_login_attempts` → 锁定账户
3. 锁定期间尝试登录 → 提示"账户已锁定，请 X 分钟后重试"
4. 锁定时间到期 → 自动解锁，重置失败计数
5. 登录成功 → 重置失败计数和锁定状态

> ⚠️ **管理员豁免**：管理员登录失败不计入锁定机制

### 20.2 密码复杂度策略

| 配置项 Key                   | 默认值 | 范围 | 说明         |
| ---------------------------- | ------ | ---- | ------------ |
| `password_policy_enabled`    | false  | —    | 策略总开关   |
| `password_min_length`        | 8      | 8-32 | 最小密码长度 |
| `password_require_uppercase` | true   | —    | 需要大写字母 |
| `password_require_lowercase` | true   | —    | 需要小写字母 |
| `password_require_digit`     | true   | —    | 需要数字     |
| `password_require_special`   | true   | —    | 需要特殊字符 |

**策略未启用时**：仅要求最少 6 个字符

**策略启用时**：必须满足所有启用的规则

### 20.3 管理员密码重置（CLI）

```bash
flask reset-admin-password
```

**交互流程**：

1. 输入管理员用户名
2. 输入新密码（输入时不显示明文）
3. 确认新密码
4. 系统校验密码是否符合策略要求

**注意事项**：

- 重置后 `must_change_password` 标记为 `True`
- 该操作会记录安全审计日志
- 仅限 `role='admin'` 的用户

### 20.4 CSRF 防护

- 系统使用 Flask-WTF `CSRFProtect` 进行全局 CSRF 防护
- 所有 POST 请求必须携带 `csrf_token`
- 模板中通过 `{{ csrf_token() }}` 注入令牌

### 20.5 文件上传安全

**扩展名白名单**：PNG / JPG / JPEG / GIF / SVG

**Magic Bytes 校验**：

- 系统读取文件头部字节验证真实文件类型
- 防止扩展名伪造

**SVG 安全检测**：
拒绝包含以下危险内容的 SVG 文件：

- `<script>` 标签
- 事件处理器属性（`onclick=`、`onerror=`、`onload=` 等）
- `<iframe>`、`<object>`、`<embed>` 标签
- `javascript:` 协议
- `<link>` 标签的 `href` 属性

**文件大小限制**：

```
MAX_CONTENT_LENGTH = 16MB
```

### 20.6 XSS 防护

- 聊天消息使用 `markupsafe.escape` 进行 HTML 转义
- Jinja2 模板默认开启自动转义
- 用户输入在存储和展示时均经过处理

### 20.7 请求限流

| 接口     | 限流规则                        | 说明         |
| -------- | ------------------------------- | ------------ |
| 注册接口 | 同一 IP 每 60 秒最多 3 次       | 防止批量注册 |
| 登录接口 | 同一用户名/IP 每 60 秒最多 5 次 | 防止暴力破解 |

**超限响应**：HTTP 429 + `Retry-After` 头 + 提示信息

### 20.8 生产环境密钥安全

**不安全密钥黑名单**：

```
your-secret-key-change-in-production
dev
development
secret
change-me
flask-secret-key
test
```

- **生产环境**：使用默认/不安全密钥 → 直接抛出 `RuntimeError` 拒绝启动
- **开发环境**：使用默认密钥 → 仅发出警告

**生成安全密钥**：

```bash
# 方法 1：使用 openssl
openssl rand -hex 32

# 方法 2：使用 Python
python -c "import secrets; print(secrets.token_hex(32))"
```

### 21.3 AI 任务架构

**异步任务处理**：

- 使用 `threading.Thread` 后台执行 AI 分析
- `AIAsyncTask` 模型跟踪任务状态（pending/running/completed/failed）
- 进度实时更新（0% → 10% → 90% → 100%）
- 切换页面不丢失分析状态

**优势**：

- AI 分析耗时较长，异步执行不阻塞用户体验

- 进度实时更新让学生了解分析进展

### 21.4 多 LLM 编排器

**调用流程**：

```
主服务商 → 失败 → 自动切换备用 → 失败 → 继续切换
```

**关键特性**：

- 每个服务商支持多次重试（默认 2 次）+ 重试延迟（默认 3 秒）
- 自动记录 Fallback 事件，追踪切换原因
- Token 预算控制：每个服务商可设置每日上限，超预算自动跳过
- 全链路日志便于追踪和优化

## 24. 常见问题 FAQ

### 学生常见问题

**Q: 为什么我的账户被锁定了？**

→ 连续输错密码 5 次（默认）后账户会被锁定 30 分钟（默认），这是安全保护机制。等待锁定时间结束即可重新登录。

**Q: 怎么修改密码？**

→ 登录后进入"个人资料"页面，在密码修改区域输入原密码和新密码。

**Q: 答错的题目在哪里看？**

→ 错题本会自动收录你答错的题目。进入"错题本"页面即可查看。

**Q: 为什么有些题目的选项有图片？**

→ 题目支持图文混排，选项可以同时包含文字和图片。仔细查看图片内容后再作答。

**Q: 排行榜怎么计算的？**

→ 排行榜支持多种维度：总积分、正确率、胜场数、连续天数等。可在排行榜页面切换查看。

**Q: 段位怎么升级？**

→ 积分达到对应段位的门槛时自动晋升。例如积分达到白银门槛，段位自动从青铜升为白银。

**Q: 对战中对方退出怎么办？**

→ 游戏继续进行，对方成绩按已答部分计算。你可以继续答题完成游戏。

**Q: 掉线了还能继续答题吗？**

→ 可以！重新登录后系统会检测到进行中的游戏，从断点位置继续答题。

**Q: 为什么看不到某些学科/章节的题目？**

→ 可能该学科/章节暂无题目，或已被管理员停用。请更换其他学科/章节。

**Q: 错题复习答对了为什么还在错题本？**

→ 需要连续正确 3 次（默认）才会标记为"已掌握"并从复习列表移除。偶尔答对一次不代表真正掌握。

**Q: 再来一局邀约超时了怎么办？**

→ 再来一局邀约 15 秒超时自动取消。你可以重新发起邀约。

**Q: 四人挑战人不够怎么办？**

→ 等待其他人加入房间。房间默认 20 分钟过期，过期后可创建新房间。

**Q: 多选题少选了算对吗？**

→ 不算。多选题必须选中所有正确选项才算对，少选、多选、错选均判为错误。

**Q: 填空题输入空格影响判分吗？**

→ 不影响。系统会自动忽略答案首尾的空格，所以" 北京 "和"北京"效果相同。

### 教师常见问题

**Q: 教师看不到某功能菜单？**

→ 检查该教师的权限配置。进入用户管理 → 编辑该教师 → 确认对应权限开关已开启。

**Q: 题库导入失败？**

→ 检查以下项目：

1. Excel 格式是否符合模板要求
2. 必填字段（学科、题型、题干、正确答案）是否完整
3. 题型值是否为 `single`/`multiple`/`judge`/`fill`
4. 难度值是否为 1-4 的整数

**Q: 如何批量导入学生？**

→ 进入 **学生管理 → 学生导入** 页面，上传符合格式的 Excel 文件。导入的学生默认角色为 `student`，首次登录强制修改密码。

### 管理员常见问题

**Q: 忘记管理员密码怎么办？**

→ 在服务器命令行执行 `flask reset-admin-password`，按提示输入用户名和新密码即可重置。重置后首次登录会强制修改密码。

→ 如需恢复默认管理员账号（admin/admin123），可使用数据备份页面的"重置系统"功能。注意：此操作会清空所有数据！

**Q: 数据库文件过大？**

→ 使用备份+重置+恢复流程：先创建备份 → 重置系统数据 → 仅恢复必要数据

**Q: 上传文件被拒绝？**

→ 检查：

1. 文件扩展名是否在白名单内（PNG/JPG/JPEG/GIF/SVG）
2. 文件大小是否超过 16MB
3. SVG 文件是否包含危险内容

**Q: WebSocket 连接失败？**

→ 检查：

1. `SOCKETIO_CORS_ALLOWED_ORIGINS` 配置是否包含客户端地址
2. 网络防火墙是否允许 WebSocket 连接
3. 浏览器是否支持 WebSocket

**Q: 生产环境启动报错 "RuntimeError: Insecure SECRET_KEY"？**

→ 在环境变量或 `.env` 文件中设置强随机的 `SECRET_KEY`，例如：

```bash
SECRET_KEY=$(python -c "import secrets; print(secrets.token_hex(32))")
```

**Q: AI 功能连接失败？**

→ 检查：

1. LLM_ENCRYPTION_KEY 是否配置
2. AI 服务商配置是否正确
3. API Key 是否有效
4. 网络是否能访问 AI 服务

### 运维常见问题

**Q: 服务无法访问？**

```bash
# 1. 检查服务状态
docker compose ps

# 2. 查看应用日志
docker compose logs --tail=50 app

# 3. 检查端口占用
sudo lsof -i :80

# 4. 检查防火墙
sudo ufw status
sudo ufw allow 80/tcp
```

**Q: 数据库密码认证失败？**

→ 查看 PostgreSQL 实际使用的密码：

```bash
docker compose exec db env | grep POSTGRES_PASSWORD
```

将 `.env` 中的密码改为上面显示的密码，然后重启应用。

**Q: 如何升级系统？**

→ 按以下步骤操作：

1. 备份数据库
2. 拉取最新代码：`git pull`
3. 检查配置：`./scripts/setup-env.sh`
4. 重建服务：`docker compose up -d --build`
5. 执行迁移：`docker exec studyclash-app flask db upgrade`

## 26. 术语表

| 术语         | 解释                                                |
| ------------ | --------------------------------------------------- |
| 房间码       | 6 位随机码（大写字母+数字），用于加入对战房间       |
| 段位         | 根据积分划分的等级，共 24 级（8 大段位 × 3 小段位） |
| 连续签到     | 每天登录累计的天数，断签重置为 1                    |

| 断点续答     | 掉线后从上次位置继续答题，不会丢失进度              |
| WebSocket    | 实时通信技术，用于对战中的即时交互                  |
| CSRF Token   | 安全令牌，防止表单伪造攻击                          |
| Magic Bytes  | 文件头部标识字节，用于验证上传文件的真实类型        |
| 滑动窗口限流 | 基于时间窗口的请求频率控制，防止恶意请求            |
| Fernet 加密  | 对称加密算法，用于加密存储 API Key                  |
| LLM          | 大语言模型（Large Language Model）                  |
| Token        | LLM 计费单位，表示文本片段的数量                    |
| 遗忘曲线     | 艾宾浩斯提出的记忆遗忘规律曲线                      |
| 异步任务     | 后台执行的任务，不阻塞用户界面                      |
| 编排器       | 管理多个 LLM 服务商调用顺序和降级策略的组件         |
| 知识追踪     | 追踪学生对知识点的掌握状态变化                      |
| 变式题       | 基于原题生成的相似题型，用于巩固知识点              |
